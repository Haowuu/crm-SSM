1、删除活动
    需求分析：
            用户在市场活动主页面,选择要删除的市场活动,点击"删除"按钮,弹出确认窗口;
          	用户点击"确定"按钮,完成删除市场活动的功能.
          	*每次至少删除一条市场活动
            *可以批量删除市场活动
          	*删除成功之后,刷新市场活动列表,显示第一页数据,保持每页显示条数不变
          	*删除失败,提示信息,列表不刷新
    业务实现：
        1）给删除按钮绑定单击事件，验证选择的活动条数大于0
        2）弹出确认框
        3）获取选择的活动的id，把其拼成一个参数的字符串：id=xxx&id=xxx...
        4）发起ajax请求，将字符串参数传入，返回的数据中判断是否删除成功
            成功：刷新列表
            失败：弹窗提醒，不刷新
        5）controller接收参数，参数为一个数组
        6）调用service
        7）service调用dao
        8）编写sql语句，
        9）controller接收返回数据，判断是否删除成功，往ReturnObject中写数据并返回至前端

2、修改市场活动
    需求分析：
        用户在市场活动主页面,选择要修改的市场活动,点击"修改"按钮,弹出修改市场活动的模态窗口;
        	用户在修改市场活动的模态窗口填写表单,点击"更新"按钮,完成修改市场活动的功能.
        	*每次能且只能修改一条市场活动
        	*所有者 动态的
        	*表单验证(同创建)
        	*修改成功之后,关闭模态窗口,刷新市场活动列表,保持页号和每页显示条数都不变
        	*修改失败,提示信息,模态窗口不关闭,列表也不刷新
    业务实现：
        1）给修改绑定单击事件，验证被选中的活动条数是否为1
        2）弹出模态窗口，将原信息展示在窗口，等待修改
        3）修改完成，给更新绑定单击事件，验证各个信息填写的合法性，同创建一致
        4）验证通过后，发送ajax请求，提交更新的参数
        5）controller接收参数，封装参数至Activity对象中传递至service
        6）service调dao
        7）编写sql语句
    注意：往数据库写数据时，需要加上异常处理，防止出现问题
