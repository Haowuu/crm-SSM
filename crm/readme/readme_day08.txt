第八天项目进度

1、删除市场活动备注
    需求分析：
        用户在市场活动明细页面,点击"删除"市场活动备注的图标,完成删除市场活动备注的功能.
        	*删除成功之后,刷新备注列表
        	*删除失败,提示信息,备注列表不刷新
    说明：备注往后的功能实现是在创建备注后返回当条备注进行追加的方式实现的，不是插入成功后查询所有备注，删除原有备注的基础上进行循环展现的，
            如果以list方式的话，无法判断当条记录时修改还是未修改的，代码实现繁琐

    代码实现：
        1》给删除超链接的图标创建一个新属性，用于保存/绑定当条备注的id，另外还需要再创建一个新的属性，用于和修改的超链接做区分
        2》给这个备注所在div起一个不重复的id值，可以使用备注的id加上前缀进行区分，用于删除这整个div
        3》通过on函数选择到点击的备注，设置单击事件，获取当条备注的id，用以发送
        4》发起ajax请求，传递id，取删除这个备注
        5》controller接收请求：
            1、接收参数，调用service
            2、接收返回值，根据返回值判断json的内容
            3、因为修改了数据库数据，所以要加try..catch..
            4、返回json
        6》service接收
        7》mapper接收，编写删除的SQL语句
        8》前端接收到返回json，根据内容判断处理方案
            成功：找到这个备注的所在整个div，用remove()删除这个备注
            失败：弹窗提醒


2、修改市场活动备注
    需求分析：
        用户在市场活动明细页面,点击"修改"市场活动备注的图标,弹出修改市场活动备注的模态窗口;
        	用户在修改市场活动备注的模态窗口,填写表单,点击"更新"按钮,完成修改市场活动备注的功能.
        	*备注内容不能为空
        	*修改成功之后,关闭模态窗口,刷新备注列表
        	*修改失败,提示信息,模态窗口不关闭,列表也不刷新

    代码实现：
        1》给修改超连接创建新的属性来保存备注的id，再创造一个新的属性，用来区分和删除的区别
        2》给超连接绑定点击时间，单击链接后，将备注里的信息内容填充到模态窗口中，将此条备注的id写入到模态窗口中的隐藏域中
        3》展开模态窗口
        4》在填写完修改的备注以后，点击更新按钮，获取文本域的备注值和隐藏域中的id值，判断备注是否为空，不为空则发送ajax请求
        5》jsp接收到返回数据，判断是否更改成功，成功则将本条备注中的标签重新刷新一下（以写入html()的方式）
        6》controller接收请求
            1、封装数据，在controller中将修改人修改日期及修改标记封装进对象中
            2、传递参数值service中
            3、接收返回值，判断并返回对应数据，修改数据要进行异常处理
        7》service接收传递参数值mapper
        8》mapper接收，写更新的SQL语句
